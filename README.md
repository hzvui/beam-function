Данный скрипт представляет собой **статический вспомогательный класс `CameraCast`** (не являющийся компонентом UNIGINE), который обеспечивает централизованную функциональность для определения объекта, на который направлен луч (рейкастинг), управления подсветкой (outline) и хранения состояния «захваченного» объекта. Он служит связующим звеном между системами ввода (VR-контроллеры или мышь/клавиатура) и визуальной обратной связью, а также предоставляет другим компонентам актуальную информацию о текущем выбранном объекте.

**Основные возможности:**

- **Определение объекта под лучом:**  
  Метод `GetObject()` выполняет бросание луча:
  - В **VR** – из позиции правого контроллера по его направлению (красная линия).
  - На **ПК** – из камеры игрока (`VRPlayer`) вперёд на 100 метров.
  Возвращает первый пересечённый объект типа `Object`.

- **Управление подсветкой (outline):**  
  Методы `SetOutline()`, `SetOutlineTest()`, `SetOutlineWithColor()` и `ResetOutline()` включают/отключают эффект подсветки на всех поверхностях объекта через материал (состояние `auxiliary` и параметр `auxiliary_color`). Это позволяет визуально выделять выбранные или интерактивные объекты.

- **Состояние «захвата» (drugged):**  
  Статические поля `drugged` и `drugObject` хранят информацию о том, удерживает ли пользователь в данный момент объект (через `SetDrugged()`). Это используется компонентом `DrugObject` для блокировки выделения во время манипуляции.

- **Получение трансформаций:**  
  Методы `GetOldWorldTransform()` и `GetIWorldTransform()` возвращают матрицы преобразования текущего источника луча (контроллера или камеры) – полезно для позиционирования захваченного объекта относительно инструмента.

- **Интеграция с `GetterModels` и `VRPlayer`:**  
  Класс инициализируется с экземпляром `GetterModels`, из которого получает ссылку на ноду игрока (`VRPlayer`), необходимую для работы камеры на ПК.

**Зависимости:**  
Для корректной работы требуется наличие на сцене компонента `VRPlayer` (прикреплённого к ноде игрока) и правильная настройка материалов с поддержкой состояний `auxiliary` и `auxiliary_btw`.

Благодаря своей статической природе `CameraCast` используется множеством компонентов (`SelectObject`, `DrugObject`, `QuizController` и др.) как единый источник данных о рейкастинге и подсветке, что упрощает архитектуру проекта и избегает дублирования кода.
